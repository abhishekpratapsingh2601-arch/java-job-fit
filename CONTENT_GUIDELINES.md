# JavaJobFit Content Guidelines

JavaJobFit content should help Java developers make better resume decisions without hype.

## Writing Rules

- Write for Java developers, Spring Boot developers, backend engineers, and Java freshers.
- Prefer concrete examples over vague advice.
- Use honest beta wording.
- Do not claim guaranteed interviews, recruiter responses, ATS success, job offers, or hiring outcomes.
- Do not add fake stats, fake testimonials, fake logos, fake users, or fake company claims.
- Do not copy content from other sites.
- Each SEO page must have unique useful content and a clear search intent.
- Use simple language.
- Add the CTA naturally: Analyze my Java resume.
- Keep the privacy promise visible: raw resume text and raw job descriptions are not stored.

## Good Content Pattern

Use this structure for Java resume pages:

1. Clear H1 matching the search intent.
2. Short useful intro.
3. Who the page is for.
4. Java/Spring Boot/backend keywords.
5. Good vs bad resume bullet examples.
6. Common mistakes.
7. How JavaJobFit helps.
8. Related Java resume resources.
9. Privacy note.

## Content Quality Bar

A page is not ready if it could apply to any generic job seeker. It should mention relevant Java/backend details such as Spring Boot, REST APIs, Hibernate/JPA, SQL, testing, Docker, Kafka, microservices, production support, or remote collaboration when those topics match the page.
